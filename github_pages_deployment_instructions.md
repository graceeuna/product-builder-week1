# GitHub Pages 배포 안내

현재 디렉토리의 내용을 웹으로 배포하는 방법을 안내해 드리겠습니다.

코드가 GitHub에 호스팅되어 있으므로, 가장 빠르고 쉬운 방법은 GitHub Pages를 활용하는 것입니다. GitHub Pages는 HTML, CSS, JavaScript로 구성된 정적 웹사이트를 호스팅하는 데 적합합니다.

다음 단계를 따라 GitHub Pages를 활성화하고 배포를 진행해 주세요:

1.  **GitHub 리포지토리로 이동합니다:**
    웹 브라우저에서 사용자님의 GitHub 리포지토리 페이지로 이동합니다. (예: `https://github.com/graceeuna/product-builder-week1`)

2.  **'Settings' 탭을 클릭합니다:**
    리포지토리 페이지 상단에 있는 'Settings' 탭을 클릭합니다.

3.  **좌측 사이드바에서 'Pages'를 찾습니다:**
    왼쪽 사이드바에서 'Code and automation' 섹션 아래에 있는 'Pages'를 클릭합니다.

4.  **GitHub Pages를 설정합니다:**
    *   'Build and deployment' 섹션 아래의 'Source'에서 'Deploy from a branch'를 선택합니다.
    *   'Branch' 드롭다운 메뉴에서 `main` 브랜치를 선택합니다.
    *   폴더는 기본값인 `/ (root)`를 선택합니다.
    *   'Save' 버튼을 클릭합니다.

이후 GitHub가 자동으로 웹사이트를 빌드하고 배포를 시작합니다. 배포가 완료되는 데 몇 분 정도 소요될 수 있습니다. 배포가 완료되면, 일반적으로 `https://<사용자-GitHub-아이디>.github.io/<리포지토리-이름>` 형태의 URL로 접속할 수 있습니다.

예를 들어, 사용자님의 웹사이트는 `https://graceeuna.github.io/product-builder-week1` 와 유사한 주소로 접근 가능할 것입니다.

이대로 진행하시면 웹 배포가 완료됩니다. 추가적인 도움이 필요하시면 알려주세요.
